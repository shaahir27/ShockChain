===============================
PROJECT: ShockChain Simulation Engine
===============================

1. SHORT CODE EXPLANATION
------------------------
This project simulates cascading economic ripple effects caused by a disruption
in global commodity supply (e.g., oil, grain, semiconductors).

The system models countries as nodes in a graph and trade dependencies as edges.
Each commodity is treated as a separate graph layer.

A shock (export reduction) is applied to a selected exporter node. The impact is
then propagated through the network using Breadth First Search (BFS), simulating
how disruptions cascade across countries.

Each country has commodity-specific GDP sensitivity factors, which are used to
calculate economic loss.

--------------------------------

2. INPUTS REQUIRED
------------------

User Inputs (via terminal):
- Commodity index (selected from list printed at runtime)
- Exporter index (selected from list printed at runtime)
- Export reduction percentage (e.g., 20 for 20%)

--------------------------------

Code Input Locations:
- main() function:
    - scanf() for commodity selection
    - scanf() for exporter selection
    - scanf() for reduction percentage

--------------------------------

File-based Inputs (datasets):

Located inside the "data/" directory:

1. commodities_list.txt
   - List of commodities (one per line)
   Example:
     Oil
     Grain
     Semiconductors

2. nodes.txt
   - Defines countries and GDP factors
   Format:
     name type oil_factor grain_factor semicon_factor annual_import

3. edges_<Commodity>.txt
   - Defines trade dependencies
   Example:
     edges_Oil.txt
     edges_Grain.txt
     edges_Semiconductors.txt

--------------------------------

3. OUTPUTS OF THE CODE
---------------------

1. Debug Logs:
   Shows ripple propagation step-by-step
   Example:
     [DEBUG] Saudi -> India : 10.00%

2. Final Results:
   For each country:
     - Total Impact (%)
     - GDP Loss (%)
     - Required Price Increase (%)

--------------------------------

4. RELATED DATASETS
------------------

All datasets are stored in the "data/" folder:

- commodities_list.txt
- nodes.txt
- edges_Oil.txt
- edges_Grain.txt
- edges_Semiconductors.txt

--------------------------------

5. KEY FEATURES
--------------

- Multi-layer graph (one per commodity)
- Targeted shock simulation (single exporter)
- BFS-based ripple propagation
- Commodity-specific GDP impact
- Fully data-driven design

--------------------------------

6. NOTES
--------

- The model assumes linear propagation of impact
- Real-world factors like buffering, substitution, and policy intervention
  are not included but can be added in future versions
